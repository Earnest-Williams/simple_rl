# Asset pipeline

This document is the source of truth for repository asset ownership, generated
font artifacts, and update expectations. It addresses the generated-asset policy
follow-up from `docs/REPOSITORY_AUDIT.md`.

## Font asset ownership

| Path | Role | Source of truth | Update policy |
| --- | --- | --- | --- |
| `fonts/Cheepicus_12x12.png` | Runtime tileset/font image. | Authored binary asset. | Keep checked in; update only when the runtime tileset intentionally changes. |
| `fonts/classic_roguelike_white.png` | Source atlas for the classic roguelike tile family. | Authored binary asset. | Keep checked in; regenerate derivatives after atlas changes. |
| `fonts/classic_roguelike_preview.png` | Human review preview/contact sheet for the classic roguelike atlas. | Derived from `fonts/classic_roguelike_white.png`. | Keep checked in as a stable review artifact; update with atlas or glyph-map changes. |
| `fonts/classic_roguelike_sliced/` | Runtime PNG tiles consumed as a directory by rendering/tooling. | Derived from the classic roguelike atlas. | Keep checked in for runtime convenience; regenerate after atlas slicing changes. |
| `fonts/classic_roguelike_sliced_svgs/` | Runtime SVG tiles consumed as a directory by rendering/tooling. | Derived from the classic roguelike atlas/vector conversion workflow. | Keep checked in for runtime convenience; regenerate after atlas or SVG conversion changes. |
| `fonts/glyph_name_chart.md` | Curated glyph naming chart and notes. | Authored markdown table. | Edit by hand when glyph names, confirmations, or notes change. |
| `fonts/glyphs.yaml` | Runtime glyph metadata loaded by `engine/glyphs.py`. | Generated from `fonts/glyph_name_chart.md` and sliced tile directories. | Keep checked in; regenerate with `python scripts/generate_glyphs.py` after chart or sliced asset changes. |
| `fonts/glyphs_report.txt` | Validation/report output for glyph metadata generation. | Generated by `scripts/generate_glyphs.py`. | Keep checked in beside `fonts/glyphs.yaml` so reviewers can inspect skipped rows and coverage. |
| `fonts/tree.txt` | Static snapshot of the font directory tree. | Derived documentation snapshot. | Treat as archival/diagnostic only; do not use as a source of truth for current file presence. Prefer `rg --files fonts`. |

## Regenerating glyph metadata

Run the generator from the repository root after changing
`fonts/glyph_name_chart.md`, `fonts/classic_roguelike_sliced/`, or
`fonts/classic_roguelike_sliced_svgs/`:

```bash
python scripts/generate_glyphs.py
```

The command updates both:

- `fonts/glyphs.yaml`
- `fonts/glyphs_report.txt`

Review both generated files in the same change as the source chart or sliced
asset changes. The generator validates referenced PNG/SVG tile files before
writing metadata.

## Runtime consumers

Current runtime and tooling consumers reference asset directories or generated
metadata rather than individual sliced filenames:

- `engine/glyphs.py` loads `fonts/glyphs.yaml`.
- `engine/window_manager.py` resolves both classic roguelike sliced asset
  directories.
- `config/config.yaml` and `tools/play_from_arrow.py` default to
  `fonts/classic_roguelike_sliced_svgs`.
- `tools/lighting_fov_tool/tool_window.py` uses the sliced SVG tiles for the
  lighting/FOV tool.

Because these consumers load directories dynamically, an individual generated
PNG/SVG tile may have no direct text reference while still being part of the
runtime asset set.

## Review checklist for asset changes

Before submitting asset changes:

1. Confirm authored assets and generated derivatives changed together.
2. Run `python scripts/generate_glyphs.py` if glyph chart or sliced tiles changed.
3. Inspect `fonts/glyphs_report.txt` for missing PNG/SVG warnings.
4. Update this document when ownership, generation commands, or retention policy
   changes.
5. Prefer `pyproject.toml` and `environment.yml` for dependency setup; do not add
   asset-pipeline-only dependencies to `requirements.txt`.
