# Engineering Rules & Standards

This document is the entry point for all engineering rules in the Simple RL
repository.  Detailed rules are maintained in a **single canonical file** and
automatically synced to agent-specific locations.

## Canonical locations

| Concern | Canonical file | Notes |
|---------|---------------|-------|
| LLM / contributor rules | [`docs/LLM Critical Rules.md`](./LLM%20Critical%20Rules.md) | Single source of truth |
| LLM policy copies | `.codex/AGENTS.md`, `.gemini/styleguide.md`, `CLAUDE.md` | Auto-generated by `scripts/sync_llm_policy.py` |
| Copilot instructions | `.github/copilot-instructions.md` | Extended guide for GitHub Copilot |
| Root contributor guide | `AGENTS.md` | Extended PEP 585 migration plan |
| Project config | `pyproject.toml` | Pinned tool versions (mypy, black, ruff) |

## Formatting & linting

A single canonical script handles all formatting and linting:

```bash
./tools/style/format_and_lint.sh          # whole repo
./tools/style/format_and_lint.sh path/    # specific directory
```

Sub-directory `fix.sh` scripts (in `auto/`, `pathfinding/`, `scripts/`) are
thin stubs that delegate to the canonical script.

Pre-commit hooks are configured in `.pre-commit-config.yaml` using the same
pinned tool versions from `pyproject.toml`.

## Tuning constants

Cross-cutting numeric constants are centralized in `common/tuning.py`.
Subsystem-specific constants remain in their own `constants.py` modules.

To find potential duplicates:

```bash
python scripts/find_tuning_dupes.py
```

See [`common/tuning.py`](../common/tuning.py) for the current set of
centralized values and their documentation.

## Key rules summary

1. **Python 3.11+** — PEP 604 unions, PEP 585 generics, `pathlib.Path`
2. **Determinism** — `GameRNG` only; no `random` or NumPy RNG in game logic
3. **Type safety** — `mypy --strict`; `typing.Final` for constants
4. **Performance** — Polars (not Pandas); Numba for hot paths; vectorize
5. **Formatting** — `black` at 88 chars; `ruff` for linting
